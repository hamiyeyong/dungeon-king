# Spec: 입력 시스템 (Input System)

> Status: 작성 중
> Priority: Phase 1 MVP (층 이동보다 먼저 구현)

---

## 개요

모바일 터치 기반 이동. 이동하고 싶은 방향/타일을 탭하면 그쪽으로 1칸 이동.
PC 에디터 테스트용 키보드 입력은 병행 유지.

## 관련 씬/노드

`Main` · `Player` · `Map`

## 의존 스펙

- [[01-map-generation]] — 타일 좌표 변환 기준

## 입력 차단 조건

```
다음 상태에서는 플레이어 입력(이동/공격) 완전 차단:
  - input_blocked = true  (페이드 전환, 게임오버 등)
  - 팝업이 열려있는 동안  (가방/제작/모닥불/확인창 등 모든 팝업)
  - _acting = true        (공격/이동 애니메이션 진행 중)
  - sleep_turns > 0       (수면 상태이상)

구현: player._unhandled_input() 상단에서 hud.is_any_popup_open() 체크
```

---

## 탭 이동 방식 (Pixel Dungeon 방식)

```
화면 탭
  └─ 탭 위치 → 월드 좌표 → 타일 좌표 변환
  └─ 탭한 타일과 플레이어 타일의 방향 계산
       └─ 인접 타일 (상하좌우) → 해당 방향으로 1칸 이동
       └─ 같은 타일 탭           → 대기 (1턴 소모)
       └─ 대각선 or 먼 타일 탭   → 더 가까운 축 방향으로 이동
```

### 방향 계산 예시
```
플레이어 (5, 5) 기준으로 탭한 타일:
  (5, 4) → 위
  (6, 5) → 오른쪽
  (7, 3) → 오른쪽 or 위 (dx > dy이면 오른쪽, dy > dx이면 위)
```

## 구현

```
InputEventScreenTouch (모바일)
InputEventMouseButton (PC 테스트용)
  └─ 좌클릭 위치 → tile_pos 계산
  └─ player._try_move(direction)
```

**카메라 오프셋 보정 필요**
- 화면 좌표 → 월드 좌표: `get_global_mouse_position()` 사용

## ~~미결정 사항~~

- [x] 대각선 탭 → 두 축을 순서대로 2턴 연속 이동
  - 예: 플레이어 (5,5), 탭 (7,3) → 오른쪽 1칸 이동(1턴) → 위 1칸 이동(1턴)
  - dx > dy면 수평 먼저, dy > dx면 수직 먼저
  - 2번 모두 턴 소비 (배고픔 2 감소, 몬스터 2번 행동)
- [x] 탭 홀드 연속 이동 → MVP 범위 제외 (홀드 감지 복잡도 대비 효용 낮음)
