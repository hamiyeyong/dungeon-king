# Spec: 층 이동 (Floor Transition)

> Status: 작성 중
> Priority: Phase 1 MVP

---

## 개요

플레이어가 계단 타일을 밟으면 다음 층으로 이동한다.
5층 클리어 시 게임 클리어 화면을 표시한다.

## 관련 씬/노드

`Main` · `Map` · `Player` · `HUD`

## 의존 스펙

- [[01-map-generation]] — 맵 재생성 및 계단 배치
- [[02-monster-combat]] — 층별 몬스터 수/HP 스케일

---

## 동작 방식

```
플레이어가 계단 타일로 이동
  └─ 확인 팝업 표시: "다음 층으로 이동하시겠습니까?"
       └─ [확인]  → 현재 층 < 5  → 새 맵 생성 + 층 번호 +1 + 플레이어 재배치
                   현재 층 = 5  → 게임 클리어 화면
       └─ [취소]  → 팝업 닫고 계속 플레이
```

### 층 이동 시 유지되는 것
- 플레이어 HP
- 배고픔
- (추후) 인벤토리 아이템

### 층 이동 시 리셋되는 것
- 맵 (새로 절차적 생성)
- 몬스터 배치

---

## 씬/노드 구조

```
Main
  ├─ Map.generate(floor_num)  ← 층 번호 넘겨서 난이도 조절 가능하게
  └─ Player.on_floor_changed()
```

## 층 이동 연출

팝업에서 [확인] 선택 시:
1. 화면 페이드 아웃 (검정, 약 0.4초)
2. 새 맵 생성 + 플레이어 재배치
3. 화면 페이드 인

구현: `ColorRect` (전체 화면 검정) + `Tween` 으로 알파 0→1→0

---

## 층별 난이도 스케일

- **몬스터 수**: `3 + floor_num * 2` 마리 (기존 스펙 유지)
- **몬스터 HP**: `6 + (floor_num - 1) * 3` (1층=6, 2층=9, 3층=12 ...)
- ATK/DEF는 MVP에서 고정 (ATK 3, DEF 0)
