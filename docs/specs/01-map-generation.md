## 개요

절차적 생성 방식. 층마다 새로운 맵을 생성한다.
방(Room) 기반 던전 구조이며, 복도로 연결된다.

## 맵 구조

```
셀 타입
  FLOOR           — 이동 가능한 바닥
  WALL            — 이동 불가 벽
  STAIRS          — 다음 층 계단 (위로만, 한번 올라가면 못 내려옴)
  CHEST           — 보물상자 (닫힘)
  CHEST_OPEN      — 보물상자 (열림)
  DOOR            — 문 (닫힘) ← 추가 예정
  DOOR_OPEN       — 문 (열림) ← 추가 예정
  GRASS           — 수풀 (공격하면 아이템 드롭)
  JAR             — 항아리 (공격하면 랜덤 결과)
  JAR_OPEN        — 항아리 (파괴됨)
  TRAP            — 함정 (밟으면 발동)
  CAMPFIRE        — 야영지 (불 켜짐)
  CAMPFIRE_OUT    — 야영지 (불 꺼짐)
  WHITE_CAULDRON  — 연금술 흰 솥 ← 추가 예정
  BLACK_CAULDRON  — 연금술 검은 솥 ← 추가 예정
  MAGIC_WELL      — 이상한 우물 ← 추가 예정
```

## 맵 오브젝트

### 🌿 수풀 (Grass)
```
배치    바닥 타일의 약 2.5% 무작위 배치
상호작용  공격하면 소멸 + 아이템 1개 드롭
드롭 테이블
  나뭇가지  30%
  약초      25%
  돌        20%
  빈병      15%
  물약       7%  (미확인 상태)
  주문서     3%  (미확인 상태)
특이사항  밟아도 아무 일 없음. 공격해야 수집 가능
```

### 🏺 항아리 (Jar)
```
배치    바닥 타일의 약 2% 무작위 배치
상호작용  공격하면 파괴(JAR_OPEN) + 결과 발동
결과
  40%  골드 또는 아이템 드롭 (천/나무/물약 등)
  30%  빈 항아리 (아무것도 없음)
  20%  몬스터 1마리 즉시 소환
  10%  저주 상태이상 부여
```

### 🔥 야영지 (Campfire / Campfire_Out)
```
배치    각 층 룸 중앙에 1개 (CAMPFIRE 상태로 시작)
점화 재료  꺼진 야영지(CAMPFIRE_OUT) → 나뭇가지 ×1 또는 횃불 ×1 소모 → 재점화
기능    [요리] 식량 → 익은 식량, 생고기 → 구운 고기 (여러 번 가능, 불 유지)
        [휴식] 피로도 초기화 + HP +5 (1회만, 이후 CAMPFIRE_OUT)
```

### ⚗️ 연금술 솥 (White/Black Cauldron)  ← 추가 예정
```
배치    각 층 1개, 흰 솥 또는 검은 솥 중 하나 랜덤 배치
상호작용  인접 이동 시도 → 물약 제작 팝업
기능    [흰 솥] 빈병 + 재료 2개 → 물약 (발현도 높은 재료 기준)
        [검은 솥] 빈병 + 재료 2개 → 물약 (발현도 낮은 재료 기준)
특이사항  사라지지 않음 (무한 사용 가능)
```

### 🌀 이상한 우물 (Magic Well)  ← 추가 예정
```
배치    층당 0~1개 (낮은 확률)
상호작용  아이템을 던지면 변환
결과
  50%  같은 종류의 다른 아이템으로 변환
  30%  강화 +1
  15%  축복 부여
   5%  아이템 소멸 또는 저주
```

### 🪤 함정 (Trap)
```
배치    각 층 랜덤 2~4개 (복도에 주로 배치)
상태    미발각 → 발각 → 발동 후 비활성
외관    미발각: 바닥처럼 보임 (FOV 내 가까이 오면 아이콘 표시)
        발각됨: 위험 아이콘 표시
상호작용  밟으면 즉시 발동. 1턴 소모

함정 종류:
  독가시 함정  HP -5 + 독 상태이상 (POISON_TURNS)
  화염 함정   HP -8 + 화염 상태이상 (FIRE_TURNS) + 인접 1칸 AOE
  텔레포트 함정 플레이어를 층 내 랜덤 위치로 이동
```

발각 조건:
- 인접(거리 1)하면 자동 발각 (발동 안 함, 아이콘만 표시)
- 횃불 보유 시 시야 확장으로 더 일찍 발각 가능
- 식별 주문서 사용 시 층 전체 함정 발각

### 📦 보물상자 (Chest)
```
배치    층당 최소 1개, 최대 2개 보장
상호작용  플레이어가 밟으면 오픈
내용물
  슬롯 A (필수)  포션 or 음식
  슬롯 B (필수)  장비 or 재료
```

## 문 시스템 (Door)

```
방 입구마다 문 배치 (선택적)
상태    닫힘 → 열림 (한번 열면 유지)
상호작용  문에 인접한 상태에서 이동키 → 1턴 소모하며 개방
시야    닫힌 문 → 너머 시야 차단
        열린 문 → 시야 통과 가능
몬스터  인간형만 문 열 수 있음 (추후)
```

## 시야 시스템 (Fog of War)

```
전체 맵    기본적으로 어두움 (검은 오버레이)
시야 반경  플레이어 중심 N칸만 밝혀짐

기본 시야   반경 4칸
횃불 보유   반경 7칸 (인벤에 횃불 있을 때)

닫힌 문     시야 차단 (문 너머 안 보임)
열린 문     시야 통과
```

### 탐색된 영역
- 한 번 밝혀진 칸은 어둡게 표시 (완전 검정 아님, 흐릿하게)
- 현재 시야 내: 완전히 밝음
- 과거 탐색 영역: 흐릿하게 (몬스터 위치 미표시)
- 미탐색: 완전히 어둠

## 층 이동

```
계단 (STAIRS) — 마지막 방 중앙
  밟으면 팝업 표시:
    "다음 층으로 올라갈까요?\n(다시 돌아올 수 없습니다)"
  확인 → 페이드 아웃 → 다음 층 생성 → 페이드 인
  취소 → 팝업 닫힘, 몬스터 턴 진행
```

한 방향(위)으로만. 한 번 올라가면 이전 층으로 되돌아갈 수 없음.

### 층 이동 시 유지
- 플레이어 HP, MP
- 배고픔, 피로도
- 인벤토리 아이템
- 장착 장비

### 층 이동 시 리셋
- 맵 (새로 절차적 생성)
- 몬스터 배치

## 층별 난이도 스케일

- **몬스터 수**: `3 + floor_num * 2` 마리
- **몬스터 타입**: 층마다 다양해짐 (02-monster-combat.md 참조)
